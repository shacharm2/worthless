## Implementation Decisions

### Naming: lock/unlock vs enroll
- **`lock`/`unlock` in all user-facing docs.** "Enrollment" stays for internal/architecture docs only (accurate at protocol level).
- PRD stays as-is (historical document). Add glossary note mapping enroll→lock for future readers.
- `enroll_stub.py` keeps its internal module name — no churn on internal names.
- **Rename `x-worthless-alias` → `x-worthless-key`** across all source, tests, and docs. One atomic commit. Pre-release, zero users, zero cost. "key" parallels `x-api-key`, immediately clear for agents and developers.

### Port: 8787 canonical
- **8787 is the one true port.** Already the default in `up.py`.
- Update all references: README (`8443`→`8787`), install-solo (`9191`→`8787`), everywhere else.
- `wrap` continues using OS-random ports (ephemeral, no conflicts).

### User path: wrap-first progressive disclosure
- README quickstart: `git clone` → `uv pip install -e .` → `worthless lock` → `worthless wrap python app.py` (30 seconds)
- Immediately after: inline "What just happened" section showing equivalent `worthless up` + `BASE_URL` manual steps
- Show `worthless status` output in quickstart as trust anchor — transparency is first-class for a security tool
- Show `worthless unlock` + down as "undo everything" — reversibility reduces anxiety
- "Recommended" vs "Manual" — not beginner/advanced. Both legitimate, neither condescending.
- Two install gates: (1) PyPI publish → `pip install worthless` becomes primary. (2) Domain purchase → `curl worthless.sh | sh` goes live. Until then, from-source is the only documented working path.

### README structure
Quickstart-first, in this order:
1. Title + "Make your API keys worthless to steal" tagline + 4 badges (Python 3.10+, AGPL-3.0, pre-release, tests)
2. Three-line value prop — split, proxy, budget-blown-means-key-never-forms
3. Pre-release callout (`[!NOTE]` box)
4. Quickstart using REAL command output from lock, wrap, status — no fabricated terminal demos. Inline directly in README.
5. "What just happened" progressive disclosure
6. How it works — ASCII diagram of split/proxy/reconstruct flow
7. CLI reference table
8. "Why not just..." comparison table (env vars, 1Password, Vault, provider limits)
9. "What Worthless does NOT protect against" — explicit non-goals for trust
10. Security section — link SECURITY_RULES.md, name crypto primitives (XOR, HMAC), say "no novel cryptography", honest about no audit yet
11. Pre-commit Hook section — install hook, commit with unprotected key, see it blocked (~5 lines)
12. Minimal Dev section (uv sync, pytest, ruff) + Contributing (link to CONTRIBUTING.md)

**Do NOT:** mention Rust hardening. Do NOT use centered HTML headers — plain markdown only. Do NOT claim universal --json support. Do NOT fabricate terminal output.

**Cut from README:** All install links (solo, mcp, openclaw, self-hosted, teams). They're all planned-state docs. Link to `docs/` folder for exploration.
