## Common Pitfalls

### Pitfall 1: README Word Count Bloat
**What goes wrong:** README grows beyond 1,500 words, star velocity drops per research.
**Why it happens:** Trying to explain everything. Progressive disclosure forgotten.
**How to avoid:** Target ~1,000 words. Link to docs/ for depth. "What just happened" section is READ-only, not RUN.
**Warning signs:** README exceeds 200 lines.

### Pitfall 2: Stale Port References
**What goes wrong:** Docs say 8787 but some file still says 8443 or 9191.
**Why it happens:** Port was changed in code but not all docs updated.
**How to avoid:** After port standardization, grep entire repo for 8443 and 9191. Zero results required.
**Warning signs:** Any occurrence of 8443 or 9191 outside of git history.

### Pitfall 3: Header Rename Merge Conflicts
**What goes wrong:** Renaming x-worthless-alias in same branch as README rewrite causes conflicts.
**Why it happens:** Both touch README.md and potentially docs/.
**How to avoid:** 04.1c runs on separate branch AFTER 04.1b merges. README.md in 04.1b already uses "x-worthless-key" (forward-looking). 04.1c only touches source code and tests.
**Warning signs:** Both branches modifying same files.

### Pitfall 4: Quickstart That Doesn't Actually Work
**What goes wrong:** Documented quickstart fails when a new user follows it.
**Why it happens:** Skipping manual walkthrough, assuming commands work.
**How to avoid:** Run the entire quickstart from a clean state after 04.1a fixes. Capture real output.
**Warning signs:** Any step that requires undocumented prerequisites.

### Pitfall 5: Inconsistent lock/unlock vs enroll Terminology
**What goes wrong:** User-facing docs say "enroll" somewhere, confusing users.
**Why it happens:** Internal code uses "enroll" (correct at protocol level), leaks into docs.
**How to avoid:** Grep for "enroll" in all docs/ and README. Only allowed in PROTOCOL.md and internal architecture docs.
**Warning signs:** "enroll" appearing in quickstart, install guides, or examples.
