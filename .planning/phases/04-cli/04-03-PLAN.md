  <behavior>
    - `worthless scan` (no args) runs fast mode: scans .env, .env.local, staged files in working tree
    - `worthless scan --deep` adds os.environ scan and common config files (*.yml, *.yaml, *.toml, *.json in project root)
    - `worthless scan path/to/file` scans specific file(s)
    - `worthless scan --pre-commit` mode: reads filenames from args (passed by pre-commit framework)
    - Scan shows ALL findings: PROTECTED (decoy) and UNPROTECTED
    - Protected keys shown as "PROTECTED" with masked value
    - `--show-suffix` flag reveals last 4 chars for debugging
    - `--format sarif` outputs valid SARIF v2.1.0 JSON to stdout
    - `--json` outputs JSON array of findings to stdout
    - `--quiet` suppresses all non-error output (exit code only)
    - Exit codes: 0=clean, 1=unprotected found, 2=scan error (per ESLint/Semgrep convention)
    - Context-aware action: TTY -> "Run: worthless lock", non-TTY/CI -> "See: docs.worthless.dev/ci-setup"
    - NO_COLOR respected in human-readable output
    - `worthless scan --install-hook` writes .git/hooks/pre-commit script
    - .pre-commit-hooks.yaml file created at project root for pre-commit framework
  </behavior>
  <action>
    1. Create `src/worthless/cli/commands/scan.py`:
       - `scan` command with options:
         - `paths: list[Path]` argument (optional, files to scan)
         - `--deep` flag for extended scanning
         - `--pre-commit` flag (adjusts output for hook context)
         - `--format` option: "text" (default), "sarif", "json"
         - `--show-suffix` flag for debugging
         - `--install-hook` flag (write git hook and exit)
         - `--json` flag (alias for --format json)
       - Fast mode (default): collect .env, .env.local, and any explicitly passed paths
       - Deep mode: add os.environ scanning (write to temp file, scan it), common config files
       - Load enrollment data from ~/.worthless/ (gracefully handle missing = CI mode)
       - Call scan_files() from scanner module
       - Format output: human-readable table (default), SARIF (--format sarif), JSON (--format json)
       - Human output: show file:line, provider, status (PROTECTED/UNPROTECTED), masked value
       - Summary line: "Found N keys: M protected, K unprotected"
       - Action hint: if unprotected found and TTY -> "Run: worthless lock". Non-TTY -> "See: docs.worthless.dev/ci-setup"
       - Exit code: 0 if no unprotected, 1 if any unprotected, 2 on scan error

    2. Implement --install-hook:
       - Detect .git directory (walk up from cwd)
       - Write .git/hooks/pre-commit with: `#!/bin/sh\nworthless scan --pre-commit "$@"\n`
       - chmod +x the hook file
       - If existing hook, append with comment marker (don't overwrite)

    3. Create `.pre-commit-hooks.yaml` at project root:
       ```yaml
       - id: worthless-scan
         name: Worthless API Key Scanner
         entry: worthless scan --pre-commit
         language: python
         types: [text]
         stages: [pre-commit]
         pass_filenames: true
       ```

    4. Register scan command in app.py

    5. Write tests/test_cli_scan.py:
       - Test: scan .env with real key -> exits 1, shows UNPROTECTED
       - Test: scan .env with decoy key (after lock) -> exits 0, shows PROTECTED
       - Test: scan clean file -> exits 0
       - Test: scan with --format sarif produces valid SARIF JSON
       - Test: scan with --json produces JSON array
       - Test: scan with --quiet produces no output
       - Test: --install-hook creates executable .git/hooks/pre-commit
       - Test: exit codes (0, 1, 2)
       - Test: --show-suffix reveals suffix chars
       - Test: entropy filtering (placeholder values skipped)
       - Test: --pre-commit mode processes passed filenames

         b. List enrolled aliases from shard_a_dir
