<objective>
Implement the `lock` and `unlock` commands -- the core enrollment lifecycle. Lock is the primary 90-second experience: find keys in .env, split them, store shards, rewrite .env with prefix-preserving decoys. Unlock is the manual undo.

Purpose: CLI-01 requires that `worthless enroll` (and by extension `lock`) splits keys, stores shards, and confirms protection. This is the atomic operation that makes keys "worthless to steal."

Output: Working lock/unlock commands with integration tests.
</objective>
