  <behavior>
    Gate-before-reconstruct (CRYP-05):
    - When rules engine denies a request (spend cap), reconstruct_key is never called and 402 is returned
    - When rules engine denies a request (rate limit), reconstruct_key is never called and 429 is returned
    - When rules engine passes, key is reconstructed and upstream request is made

    Transparent routing (PROX-04):
    - POST /v1/chat/completions with correct headers proxies to OpenAI upstream
    - POST /v1/messages with correct headers proxies to Anthropic upstream
    - Unknown paths return 404
    - Query parameters are stripped before adapter lookup

    Server-side reconstruction (PROX-05):
    - Reconstructed key never appears in response headers
    - Reconstructed key never appears in response body
    - x-worthless-* headers are stripped from response

    Auth (anti-enumeration):
    - Missing x-worthless-alias header returns 401 "authentication required"
    - Unknown alias returns identical 401
    - Missing shard_a (header and file) returns identical 401
    - Invalid shard (HMAC failure) returns identical 401

    Health:
    - GET /healthz returns 200 {"status": "ok"} without auth
    - GET /readyz returns 200 when keys enrolled, 503 when none

    Security:
    - Shard headers rejected over non-TLS unless WORTHLESS_ALLOW_INSECURE=1
    - Alias validated: alphanumeric, dash, underscore only (no path traversal)
    - Request headers with whitespace or null bytes in keys are rejected
    - Upstream httpx client has follow_redirects=False
    - x-worthless-* headers stripped from upstream responses
    - CORS disabled by default
  </behavior>
