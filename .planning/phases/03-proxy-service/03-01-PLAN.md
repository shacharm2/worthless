  <behavior>
    Rules engine:
    - RulesEngine with empty rules list returns None (pass-through)
    - RulesEngine with one passing rule returns None
    - RulesEngine short-circuits on first denial (returns Response, does not call subsequent rules)
    - SpendCapRule returns None when spend is under cap
    - SpendCapRule returns 402 JSONResponse when accumulated spend >= cap
    - SpendCapRule returns None when spend_cap is NULL (no cap configured)
    - RateLimitRule returns None when request rate is under threshold
    - RateLimitRule returns 429 JSONResponse with Retry-After header when rate exceeded
    - RateLimitRule uses in-memory sliding window (not SQLite)

    Metering:
    - extract_usage_openai parses usage.total_tokens from JSON response body
    - extract_usage_openai parses usage from final SSE chunk containing usage field
    - extract_usage_anthropic parses output_tokens from message_delta SSE event
    - extract_usage returns 0 when usage data is not found (no crash)

    Errors:
    - auth_error_response returns 401 with uniform "authentication required" body in OpenAI format
    - spend_cap_error_response returns 402 in provider-compatible format
    - rate_limit_error_response returns 429 with Retry-After header
  </behavior>
  <action>
    1. Create `src/worthless/proxy/__init__.py` (empty package).

    2. Create `src/worthless/proxy/config.py` with a `ProxySettings` dataclass loaded from env vars:
       - `db_path: str` (default `~/.worthless/worthless.db`)
       - `fernet_key: str` (from WORTHLESS_FERNET_KEY env var, required)
       - `default_rate_limit_rps: float` (default 100.0, from WORTHLESS_RATE_LIMIT_RPS)
       - `upstream_timeout: float` (default 120.0, from WORTHLESS_UPSTREAM_TIMEOUT)
       - `streaming_timeout: float` (default 300.0, from WORTHLESS_STREAMING_TIMEOUT)
       - `allow_insecure: bool` (default False, from WORTHLESS_ALLOW_INSECURE)
       - `shard_a_dir: str` (default `~/.worthless/shard_a`)

    3. Create `src/worthless/proxy/errors.py`:
       - `auth_error_response()` -> JSONResponse 401 with `{"error": {"message": "authentication required", "type": "authentication_error", "param": null, "code": null}}` (OpenAI format; uniform for all auth failures per CONTEXT.md anti-enumeration decision)
       - `spend_cap_error_response()` -> JSONResponse 402 in same format with message "spend cap exceeded"
       - `rate_limit_error_response(retry_after: int)` -> JSONResponse 429 with Retry-After header
       - Each function takes an optional `provider: str` param ("openai"|"anthropic") to return the matching error schema

    4. Create `src/worthless/proxy/rules.py`:
       - `Rule` Protocol with `async def evaluate(self, alias: str, request: Request) -> Response | None`
       - `RulesEngine` dataclass with `rules: list[Rule]` and `evaluate()` that iterates rules, short-circuiting on first non-None return
       - `SpendCapRule`: queries `spend_log` table via aiosqlite for sum of tokens by alias, compares against `enrollment_config.spend_cap`. Returns None if no cap set (NULL). Returns 402 if exceeded.
       - `RateLimitRule`: in-memory sliding window dict keyed by (alias, client_ip). Returns 429 with Retry-After if rate exceeded. Default 100 req/s. Uses `time.monotonic()` for window tracking.

    5. Create `src/worthless/proxy/metering.py`:
       - `extract_usage_openai(data: bytes) -> int`: parse `usage.total_tokens` from JSON body. For SSE data, scan for final chunk containing `"usage"` field.
       - `extract_usage_anthropic(data: bytes) -> int`: scan for `event: message_delta` line followed by data containing `usage.output_tokens`.
       - `async def record_spend(db_path: str, alias: str, tokens: int, model: str | None, provider: str) -> None`: INSERT into spend_log table.

    6. Extend `src/worthless/storage/schema.py`:
       - Add `spend_log` table (id, key_alias, tokens, model, provider, created_at)
       - Add `enrollment_config` table (key_alias PK, spend_cap REAL nullable, rate_limit_rps REAL default 100.0, created_at)
       - Keep existing tables unchanged

    7. Write tests FIRST (RED), then implement (GREEN):
       - `tests/test_rules.py`: Test RulesEngine pipeline, SpendCapRule, RateLimitRule as described in behavior
       - `tests/test_metering.py`: Test extract_usage_openai, extract_usage_anthropic, record_spend
