<decisions>
## Implementation Decisions

### SSE Streaming (B-1)
- Pass `stream=True` to httpx upstream call — current code buffers entire response, destroying time-to-first-token
- Restructure `secure_key` context scope so streaming reads happen inside the secure context (key material must be zeroed after stream completes, not before)
- Handle client disconnect during streaming (M-12) — ensure httpx response is properly closed

### CRYP-05 Gate Ordering (B-2)
- Split `repo.retrieve()` into two phases: fetch encrypted blob first, run rules engine gate, then decrypt only if allowed
- Current violation: Fernet-decrypts shard_b before rules engine evaluates — gate-before-reconstruct invariant requires no decryption until gate passes
- The XOR reconstruction already happens post-gate; only the Fernet layer needs reordering

### SR-01 Bytearray Compliance (B-3, M-13)
- Wrap all shard material as `bytearray` not `bytes` — enables explicit zeroing
- `shard_a` from header/file: wrap in `bytearray` immediately on receipt
- `stored.shard_b` from repository: return as `bytearray`
- Zero `shard_a`, `stored` (shard_b, commitment, nonce) after use (SR-02 / M-13)

### Async File I/O (B-4)
- Replace blocking `Path.is_file()` and `Path.read_bytes()` with `asyncio.to_thread()` equivalents
- One-line fix pattern per call site

### Error Handling & Resilience (H-1, M-9, M-10)
- Wrap httpx upstream call in try/except: `TimeoutException` → 504, `ConnectError` → 502, `HTTPError` → 502
- `record_spend` failures: log warning, do not break response (fire-and-forget is correct, but failures must be observable)
- `SpendCapRule`: fail-closed on DB errors — deny request if spend check cannot be performed

### Anti-Enumeration (H-2, M-3)
- Timing oracle: ensure constant-time path for valid vs invalid aliases (both hit the same code path, same DB query pattern)
- 404 format must match 401 format exactly — currently different structure leaks whether auth succeeded

### Spend Cap TOCTOU (H-3)
- BEGIN IMMEDIATE serializes concurrent reads to prevent stale-data races
- PoC limitation: true check-and-increment is impossible because token count is only known after upstream responds. Check (rules engine) and record (metering) remain separate operations. Two concurrent requests can both pass the cap before either records spend.
- Production fix: reserve estimated tokens at check time, reconcile after response

### Repr Redaction (H-4)
- `AdapterRequest.__repr__`: redact body/prompt content, show only metadata (model, provider, token count estimate)
- `AdapterResponse.__repr__`: redact response body, show only metadata (status, usage)

### Streaming Tests (H-5)
- Add test coverage for `_stream_with_metering` and `BackgroundTask(_record_metering)`
- Required to validate B-1 fix — must prove SSE chunks relay correctly

### SQLite Performance (H-6, H-7)
- Connection pooling: reuse connections instead of 3 open/close cycles per request
- Add index on `spend_log.key_alias` — spend cap queries scan without index currently

### Request Body Limits (M-1)
- Add request body size limit to prevent OOM from oversized payloads
- Use FastAPI/Starlette middleware or manual check

### Rate Limiter Memory (M-2)
- `defaultdict` for rate limit windows grows unboundedly with unique aliases
- Add TTL-based cleanup or use bounded dict

### Upstream Error Sanitization (M-4)
- Provider error responses relayed unmodified — could leak upstream internal details
- Sanitize error bodies before forwarding to client

### Shard A File Fallback TOCTOU (M-5)
- File existence check and read are separate operations — race condition
- Use try/except on read instead of check-then-read

### Key Material String Copy (M-6)
- `api_key.decode()` creates immutable `str` copy that can't be zeroed
- Known PoC limitation — document explicitly, plan fix for Rust reconstruction service

### Dead Code Removal (M-7, M-8)
- Remove `_uniform_401()` unused function
- Remove `dependencies.py` entirely unused module

### CORS Enforcement (M-11)
- Explicitly deny CORS rather than relying on absence of CORS headers
- Add CORSMiddleware with deny-all configuration

### Claude's Discretion
- Connection pool sizing and configuration
- Exact error message wording for sanitized upstream errors
- Rate limiter cleanup interval timing
- Request body size limit value (reasonable default)

</decisions>
