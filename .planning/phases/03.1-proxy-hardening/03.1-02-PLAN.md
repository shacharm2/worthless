  <action>
Rewrite the `proxy_request` handler in app.py. This is a large change but all modifications are to a single function. Key changes:

**B-1 SSE Streaming:**
- Use `httpx_client.build_request()` + `httpx_client.send(req, stream=True)` instead of `httpx_client.request()`
- Key material only needed for `prepare_request()` and `send()` — stream reads happen AFTER `secure_key` exits
- `_stream_with_metering()` generator uses `try/finally` with `await upstream_resp.aclose()` for cleanup on disconnect
- Non-streaming path: `await upstream_resp.aread()` then `await upstream_resp.aclose()`

**B-2 Gate Ordering (uses Plan 01 split repo):**
```python
# (f) Fetch encrypted (NO Fernet decrypt)
encrypted = await repo.fetch_encrypted(alias)
if encrypted is None: return 401

# (g) Load shard_a ...
# (h) GATE: rules engine evaluates BEFORE any decrypt
denial = await rules_engine.evaluate(alias, request)
if denial: return denial

# (i) NOW decrypt (gate passed)
stored = repo.decrypt_shard(encrypted)
```

**B-3 Bytearray + Zeroing:**
- Wrap `shard_a` from header decode in `bytearray()` immediately
- Wrap `shard_a` from file read in `bytearray()` immediately
- After request completes (in finally block): zero `shard_a[:] = b'\x00' * len(shard_a)`, zero `stored.shard_b[:] = ...`, zero `stored.commitment[:] = ...`, zero `stored.nonce[:] = ...`

**B-4 Async File I/O:**
- Replace `shard_a_path.is_file()` with `await asyncio.to_thread(shard_a_path.is_file)`
- Replace `shard_a_path.read_bytes()` with `await asyncio.to_thread(shard_a_path.read_bytes)`

**M-5 Shard A File TOCTOU:**
- Replace check-then-read with try/except: `try: raw = await asyncio.to_thread(shard_a_path.read_bytes)` catching `FileNotFoundError`

**H-1 Error Handling:**
- Wrap the httpx send call in try/except:
  - `httpx.TimeoutException` -> 504 gateway timeout
  - `httpx.ConnectError` -> 502 bad gateway
  - `httpx.HTTPStatusError` / general `httpx.HTTPError` -> 502 bad gateway
- Add `gateway_error_response(status, message)` to errors.py

**M-4 Upstream Error Sanitization:**
- Add `_sanitize_upstream_error(status_code, body, provider)` function
- For non-2xx upstream responses, sanitize the body before forwarding
- Keep status code, strip internal details from error message

**M-6 Documentation:**
- Add code comment documenting that `api_key.decode()` creates an immutable str copy as a known PoC limitation, with pointer to Rust reconstruction service for the fix

**M-9/M-10 Metering resilience:**
- Wrap `record_spend` calls in try/except, log warning on failure but don't break response

**Anti-enumeration (H-2, M-3):**
- Move the `encrypted is None` check to return uniform 401 (already uses _AUTH_BODY)
- Ensure `adapter is None` (unknown endpoint) returns 401 format, not 404 JSON — prevents endpoint enumeration
- Ensure same code path timing: always attempt shard_a loading and rules evaluation even if encrypted is None — use a dummy evaluation that takes similar time. OR simpler: since the DB lookup already happened, the timing difference is minimal. Just ensure response format is identical.

Write tests in test_proxy_hardening.py covering each behavior. Update existing test_proxy.py fixtures to use the new `fetch_encrypted`/`decrypt_shard` pattern if needed.
