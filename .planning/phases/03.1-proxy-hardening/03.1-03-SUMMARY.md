## Decisions Made
- SpendCapRule constructor changed from `db_path: str` to `db: aiosqlite.Connection` -- enables persistent connection and atomic transactions
- BEGIN IMMEDIATE used instead of BEGIN for spend cap checks -- acquires write lock immediately to prevent concurrent reads both passing
- Fail-closed pattern: any exception in SpendCapRule.evaluate returns spend_cap_error_response() -- security over availability
- RateLimitRule._windows changed from defaultdict(list) to plain dict -- avoids auto-creating entries on lookup, supports cleanup
- Body size check uses Content-Length header only -- streaming uploads without Content-Length pass through
- CORS uses allow_methods=["GET"] with empty allow_origins -- effectively denies all cross-origin requests
