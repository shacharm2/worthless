## Product Context

**Worthless** makes stolen API keys worthless. Two mechanisms:

1. **Client-side XOR split** — server never sees the full key. Server only receives Shard B, commitment, nonce at enrollment.
2. **Gate before reconstruct** — rules engine evaluates every request before Shard B is decrypted. Denied = zero KMS calls, zero reconstruction.

**Three non-negotiable architectural invariants:**

| # | Invariant | Violation means |
|---|---|---|
| 1 | Client-side splitting | Full security review required |
| 2 | Gate before reconstruction | Full security review required |
| 3 | Server-side direct upstream (reconstructed key never returns to proxy) | Full security review required |

**V1 ships:** split-key proxy for OpenAI + Anthropic, rules engine (spend_cap, rate_limit, token_budget, time_window), CLI (`enroll`, `wrap`, `scan`, `daemon`, `keys`, `status`), MCP server, self-host (Docker Compose, Railway, Render, Helm), pre-commit scan, anomaly detection (spend velocity Free tier), email + Slack alerts, OpenClaw skill.

**V1 does NOT ship:** dashboard UI, team management UI, SSO, Gemini (stretch), geo/time/model anomaly detection (Pro+), response caching, provider-managed keys.
