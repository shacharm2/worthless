  <name>Task 1: Process lifecycle module -- pipe death detection, signal forwarding, PID files</name>
  <files>

  <behavior>
    - spawn_proxy(env, port=0) starts uvicorn subprocess with proxy app, returns (Popen, actual_port)
    - spawn_proxy with port=0 uses OS-assigned random port
    - spawn_proxy with port=8787 binds to that port
    - spawn_proxy binds 127.0.0.1 only (never 0.0.0.0)
    - poll_health(port, timeout=10) polls GET http://127.0.0.1:{port}/healthz until 200 or timeout
    - create_liveness_pipe() returns (read_fd, write_fd) for pipe-based death detection
    - proxy receives WORTHLESS_LIVENESS_FD env var with read_fd
    - write_pid(pid_path, pid, port) writes PID file with PID and port
    - read_pid(pid_path) returns (pid, port) or None if stale/missing
    - check_pid(pid) returns True if process is alive
    - cleanup_stale_pid(pid_path) removes PID file if process is dead
    - forward_signal(sig, proxy, child) sends signal to both processes
    - process_group=0 used for subprocess.Popen (Python 3.11+)
    - Disable core dumps via resource.setrlimit(RLIMIT_CORE, (0, 0)) at proxy startup
  </behavior>
  <action>
    1. Create `src/worthless/cli/process.py`:

       - `disable_core_dumps()`: Set RLIMIT_CORE to (0, 0) via resource module. Catch OSError on platforms that don't support it.

       - `create_liveness_pipe() -> tuple[int, int]`: Create os.pipe(), return (read_fd, write_fd). Caller passes read_fd to proxy, keeps write_fd open. Closing write_fd = EOF on proxy side = proxy self-terminates.

       - `spawn_proxy(env: dict, port: int = 0, liveness_fd: int | None = None) -> tuple[subprocess.Popen, int]`:
         - Build uvicorn command: `[sys.executable, "-m", "uvicorn", "worthless.proxy.app:app", "--host", "127.0.0.1", "--port", str(port)]`
         - If port=0: use `--port 0` and parse actual port from uvicorn output (look for "Uvicorn running on http://127.0.0.1:NNNN")
         - Set WORTHLESS_* env vars: WORTHLESS_DB_PATH, WORTHLESS_FERNET_KEY, WORTHLESS_SHARD_A_DIR from WorthlessHome
         - If liveness_fd: add WORTHLESS_LIVENESS_FD to env, pass_fds=(liveness_fd,)
         - Use process_group=0 for new process group
         - Return (process, actual_port)

       - `poll_health(port: int, timeout: float = 10.0) -> bool`: Poll GET http://127.0.0.1:{port}/healthz every 0.3s until 200 or timeout. Use httpx.Client (sync). Return True if healthy, False if timeout.

       - `write_pid(pid_path: Path, pid: int, port: int) -> None`: Write "{pid}\n{port}" to file.

       - `read_pid(pid_path: Path) -> tuple[int, int] | None`: Read PID file, return (pid, port) or None if missing/corrupt.

       - `check_pid(pid: int) -> bool`: os.kill(pid, 0) to check if alive. Return False on ProcessLookupError/PermissionError.

       - `cleanup_stale_pid(pid_path: Path) -> bool`: Read PID, check if alive. If dead, remove file, return True (reclaimed). If alive, return False (in use).

       - `forward_signals(proxy: Popen, child: Popen | None)`: Register signal handlers for SIGINT, SIGTERM that forward to proxy and child process groups.

    2. Write tests/test_process.py:

       - Test: check_pid returns True for current process, False for nonexistent PID

       - Test: spawn_proxy + poll_health integration (spawn real proxy, check health) -- mark as integration test with timeout
       - NOTE: Signal forwarding tests are tricky in CI. Test the logic (handler registration) not the full signal delivery.

  <behavior>
    - `worthless wrap python main.py` starts proxy, injects env vars, runs `python main.py`, exits with child's exit code
    - wrap injects OPENAI_BASE_URL, ANTHROPIC_BASE_URL (all enrolled providers) pointing to 127.0.0.1:{port}
    - wrap generates per-session localhost auth token between proxy and child (WORTHLESS_SESSION_TOKEN env var)
    - wrap creates liveness pipe so proxy dies if wrap is killed
    - wrap mirrors child exit code (child exits 1 -> wrap exits 1)
    - wrap warns on stderr if proxy crashes mid-session but does not kill child
    - `worthless up` starts proxy in foreground on port 8787 (Ctrl+C to stop)
    - `worthless up -d` starts proxy in daemon mode (background)
    - `worthless up --port 9999` overrides default port
    - WORTHLESS_PORT env var overrides default port
    - up writes PID file at ~/.worthless/proxy.pid
    - up detects stale PID file and reclaims
    - up with port already in use by live proxy errors with WRTLS-107
    - Both bind 127.0.0.1 only
  </behavior>
  <action>
    1. Create `src/worthless/cli/commands/wrap.py`:
       - `wrap` command:
         - `command: list[str]` argument (the child command to run)
       - Flow:
         a. Load WorthlessHome, verify keys are enrolled (error if none)
         b. Create liveness pipe
         c. Build proxy env: WORTHLESS_DB_PATH, WORTHLESS_FERNET_KEY, WORTHLESS_SHARD_A_DIR, WORTHLESS_LIVENESS_FD
         d. Generate session token: secrets.token_urlsafe(32), set as WORTHLESS_SESSION_TOKEN in both proxy and child env
         e. Spawn proxy on port=0 (random), close read_fd in parent
         f. Poll /healthz until ready
         g. Build child env: inherit current env + {PROVIDER}_BASE_URL=http://127.0.0.1:{port} for each enrolled provider + WORTHLESS_SESSION_TOKEN
         h. Provider-to-env mapping: openai->OPENAI_BASE_URL, anthropic->ANTHROPIC_BASE_URL
         i. Spawn child with subprocess.Popen(command, env=child_env, process_group=0)
         j. Register signal handler to forward SIGINT/SIGTERM to child
         k. Wait for child to exit
         l. Close write_fd (signals proxy to die via pipe EOF)
         m. Terminate proxy, wait with timeout, kill if needed
         n. Exit with child's exit code
       - If proxy crashes during child execution: log warning to stderr, don't kill child
       - Keep wrap command synchronous (no async) per research recommendation

    2. Create `src/worthless/cli/commands/up.py`:
       - `up` command:
         - `--port/-p` option: int (default from WORTHLESS_PORT env or 8787)
         - `--daemon/-d` flag: run in background
       - Flow:
         a. Load WorthlessHome, verify keys are enrolled
         b. Check PID file: if exists and alive -> error WRTLS-107. If stale -> cleanup.
         c. Disable core dumps
         d. Spawn proxy on specified port
         e. Write PID file with proxy PID and port
         f. If daemon mode: print port info, exit 0 (proxy continues in background via setsid/double-fork or nohup pattern)
         g. If foreground: poll health, print "Proxy running on 127.0.0.1:{port}", wait for SIGINT/SIGTERM, cleanup PID file
       - Daemon mode: use os.setsid() in subprocess preexec_fn so proxy survives parent exit. Write PID file. Detach.
       - Foreground: signal handler cleans up PID file, terminates proxy
