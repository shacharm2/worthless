### 5-Tier CI Model
Wire all 5 tiers into GHA workflow files:

1. **Every push/commit (<5 min):** unit/integration tests (xdist), ruff lint, type check, static security (semgrep OSS + bandit)
2. **PR (push + extras):** coverage report with threshold gate, coverage diff comment (no regressions), test summary comment
3. **Scheduled every 3 days (~10-15 min):** mutation testing (mutmut on crypto only), extended Hypothesis, schemathesis contract tests
4. **Pre-release/tag (~15-20 min):** full mutation coverage (crypto + proxy + metering), extended fuzz campaigns, dependency audit (pip-audit), license check
5. **Manual dispatch only:** performance benchmarks, stress tests, full schemathesis sweep against live proxy

Local dev (pre-commit): ruff format + lint, basic smoke tests (optional/fast)

### Coverage Gates
- Overall threshold: **80%** (blocks PR merge)
- Per-module floors: **crypto 95%**, **proxy 85%**
- PR diff-only check: new code must not decrease coverage
- Coverage report posted as PR comment

### Security CI
- Push tier: semgrep OSS + bandit (both token-free)
- Scheduled tier: pip-audit (token-free, queries public PyPI/OSV)
- All tools run zero-secrets. No GitHub secrets needed for any tier
- When repo goes public: upgrade to semgrep Pro (free for OSS) for expanded rules

### Flaky Test Policy (Hybrid)
- Default: fail loud, no silent skips
- 1 retry as flaky detector (not a fix) — pytest-rerunfailures with max 1 rerun
- Auto-quarantine after 2 flaky runs in 7 days
- Quarantined tests run in separate non-blocking CI job
- 2-week fix-or-delete SLA on quarantined tests
- Hypothesis in CI: `derandomize=True` + `database=None`
- Each quarantine gets a beads issue filed

### Test Data Management
- Keep current mix: hardcoded fixtures for deterministic tests, Hypothesis for property tests
- No golden files — crypto outputs verified by roundtrip, not snapshot
- `fake_openai_key` in helpers.py stays as the shared test key fixture

### xdist Configuration
- Distribution strategy: **loadscope** (group by module, share fixtures per worker)
- Keep `-n auto` for worker count
- Reduces DB/tempdir isolation issues with current fixture pattern

### GHA Wiring
- Create actual workflow files matching the 5-tier model
- TESTING.md lanes map to GHA jobs
- All tools token-free — zero GitHub secrets needed

### Test Naming Conventions (Light)
- New tests: `test_<what>_<condition>_<expected>` naming pattern
- Add assertion messages to crypto and security tests only
- No mass rename of existing tests — convention applies to new/modified tests only
