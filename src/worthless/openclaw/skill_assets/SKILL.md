# Worthless skill (Phase 2.a placeholder)

Version: 0.0.0-stub

This is the Phase 2.a placeholder. Phase 3 owns the real body content
that teaches OpenClaw how to route LLM calls through the worthless proxy
(commands `worthless lock | wrap | up | status` with `--json`).

The frontmatter above is the minimum OpenClaw needs to register the skill
(verified live against `ghcr.io/openclaw/openclaw:latest`):

- `name`: skill identifier (must match folder name when installed).
- `description`: shown in `openclaw skills list`.
- `metadata.openclaw.requires.bins: [worthless]`: marks the skill as
  needing the `worthless` binary on the daemon's PATH.

Do not edit this file in production deployments — it is overwritten by
`worthless lock` whenever it diverges from the embedded asset.
