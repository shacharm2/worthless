Mechanism Your app works locally. A copied file does not work elsewhere.

Worthless replaces the real AI key in your .env with a lookalike shard. The app keeps working locally through Worthless. If .env is copied into GitHub, chat, logs, or AI-generated code, the copied key alone cannot call the provider.

Lock the file Supported AI keys become format-preserving shards.
Route locally Supported SDK traffic goes through the local Worthless proxy.
Leak less damage The copied shard alone is not enough for direct provider calls.
Positioning Scanners, vaults, gateways, and Worthless do different jobs.

Scanners find leaks, vaults store secrets, provider dashboards and gateways manage spend, and Worthless makes a copied .env AI key not a blank check.

Scanners Find exposed material. They do not change what was copied.
Vaults and dashboards Store, monitor, and govern. Local workflows still use keys.
Worthless Changes the copied value into a shard that cannot call the provider by itself.
Scope Strong claim, visible boundary.

The homepage promise is scoped to supported AI keys in local developer workflows. You still investigate leaks and follow your incident policy.

What ships now

  • Local proxy for supported AI provider SDK traffic.
  • Format-preserving .env shards and base URL routing.
  • Provider support for OpenAI, Anthropic, OpenRouter, Groq, Together, and Ollama.
  • LLM-provider key scanning, including worthless scan --install-hook.

What it does not claim

  • It does not protect a compromised machine, user account, or local proxy trust boundary.
  • Use WSL2 for Windows workflows. The native Windows path is not the supported install path.
  • It is not a generic scanner for every secret family.
  • Usage guardrails are not a provider-billed dollar ceiling.
