Bitwarden CLI was real. The package went bad.

This is worse than a typo. The package name was the real one. For a limited window, the npm path people trusted carried code that looked around for credentials.

Source: Palo Alto Networks

why it matters

Compromised CLI packages are nasty because they meet developers where secrets already are. The command sounds operational. The install path gets local access.

What gets stolen depends on what the machine exposes: environment variables, config files, session material, or credentials sitting in developer workflows. The consequence is simple: the attacker may get a working key without breaking into the provider directly.

how it got there

Reporting tied the malicious @bitwarden/cli@2026.4.0 npm package to supply-chain abuse of the delivery path, not a random lookalike package. That is the scary part: a trusted update route can become the attack.
