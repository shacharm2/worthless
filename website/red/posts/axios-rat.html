Mar 2026 / compromised release

Axios shipped a RAT for three hours.

This is the scarier version. The name was not fake. The package was trusted. For a short window, the update path itself carried attacker code.

Source: StepSecurity

why it matters

Compromised releases are different from typo traps. The name is right. The dependency graph is right. The version is the problem.

A remote access trojan is not just a leaked string. It is code that may let an attacker control or inspect the machine. That can expose keys, files, shells, and developer context.

how it got there

The attacker used the package release path. That means normal dependency updates could pull the malicious version before maintainers or scanners caught up.

attack read

maintainer path -> malicious release -> local execution

Once attacker code runs locally, secret handling has to assume the machine boundary is hot.

where worthless is boring

Worthless helps less here than people want it to. If attacker code can run as the developer and use trusted local paths, copied-key protection is no longer the main boundary.
