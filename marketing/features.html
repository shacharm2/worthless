Features

Leaked key? $0 damage.

AI agents run git add -A and push your .env to GitHub. Vaults protect the key at rest, but it's still one piece, still in memory, still leaks. And your code has to change to use them. Worthless splits the key. Leaked half is useless. No code changes. Two commands.

The key in your .env doesn't work

Your .env has one half of an XOR split. The other half lives in the proxy. Neither works alone. Someone steals your .env? They get random bytes. Mathematically useless, even with unlimited compute.

Hard spending cap

Set a dollar limit per key. Hit the cap, the proxy refuses to reconstruct. Not a soft warning. A hard stop. Your app can't override it. Even if both halves are somehow available, the budget gate fires first.

Zero code changes

worthless wrap python app.py — that's it. Your code reads environment variables exactly as before. No SDK. No imports. No config files.

Pre-commit scanning

Built-in hook catches raw API keys before they hit git. Blocks the commit, not just warns. First line of defense. The split is the second.

Fully reversible

worthless unlock — original .env restored, proxy stopped, shares deleted. Don't like it? One command and it's gone. Zero lock-in.

Self-hosted. Open source. No cloud.

Your keys never touch anyone else's server. AGPL-3.0. Zero telemetry. No account needed.
