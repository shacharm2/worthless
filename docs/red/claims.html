Proof & limits.

The short claim is strong and narrow: copied locked AI-key material alone is not enough to call the provider. Everything below is the boundary.

What we claim

Copied key alone cannot call the provider.

After locking, the value left in `.env` is a format-preserving .env shard, not the original provider key.

Proof hook

The local proxy reconstructs only on the working path. A copied shard by itself lacks the local side of the trust boundary.

Limitation

This is not protection against full same-user host compromise or attacker-controlled local code.

Your app keeps using normal SDK shapes.

Worthless uses a local proxy and provider-compatible request routing instead of making you call a new application API.

Proof hook

The website install path points to `https://docs.wless.io/install/` and the repository contains the proxy implementation.

Limitation

Some Docker, CI, or unusual SDK flows may need explicit `BASE_URL` or environment wiring.

No cloud account required.

The launch path is local-first. You do not need a hosted Worthless account to lock local AI keys.

Proof hook

The install path uses the open-source CLI and local proxy.

Limitation

Hosted team workflows are separate from the current local developer promise.

Supported platforms are macOS, Linux, and WSL.

The public platform claim includes WSL for Windows users.

Proof hook

Install docs carry the platform-specific guidance.

Limitation

This is not native Windows support.

Scanning is LLM-provider-focused.

`worthless scan --install-hook` can help catch supported AI key material before commits.

Proof hook

The scanner is scoped to LLM provider patterns and the true provider registry.

Limitation

Worthless is not a scanner replacement. Use tools like gitleaks or trufflehog for broad secret discovery.

The provider registry is explicit.

The current registry centers OpenAI, Anthropic, OpenRouter, Groq, Together, and Ollama.

Proof hook

Supported providers are represented in the product registry rather than implied by marketing chips.

Limitation

Do not read this as broad cloud, payment, source-hosting, database, or generic credential support.

What we do not claim

  • Worthless is not a general vault.
  • Worthless is not a scanner replacement.
  • Worthless is not native Windows support.
  • Worthless is not all-secret protection today.
  • Worthless is not protection against full same-user host compromise.
  • Worthless provides token-budget guardrails, not a hard spend cap.
  • Worthless does not remove the need to rotate a real provider key after a credible incident.
