Security model.

Worthless changes what copied locked AI-key material can do. It does not make the leak fine.

What is in scope

  • Supported AI keys that have been locked before exposure.
  • Local developer workflows on macOS, Linux, and WSL.
  • Accidental `.env` exposure to Git, chat, generated code, logs, screenshots, or support threads.
  • The copied key-looking value, because it is not the complete provider credential.

How requests still work

Your app routes through a local proxy. The proxy uses the stored local material to make an upstream provider call when policy allows it. A token-budget guardrail can deny before reconstruction, but that is not a hard spend cap.

Operational scope

  • No cloud account required for the local launch path.
  • The true provider registry centers OpenAI, Anthropic, OpenRouter, Groq, Together, and Ollama.
  • Scanner behavior is LLM-provider-focused. It is not a replacement for gitleaks or trufflehog.

Out of scope

  • Not a general vault.
  • Not native Windows support.
  • Not broad credential protection for non-LLM services today.
  • Not protection against full same-user host compromise.
  • Not protection if attacker-controlled code can use the same local working route.
