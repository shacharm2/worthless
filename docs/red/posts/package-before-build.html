The package ran before your build did.

A developer meant to install one thing and got something close enough to pass at a glance. The bad package ran before the app did. By the time the build started, the attacker already had a shot at the secrets in the room.

Source: Microsoft Security

why it matters

Microsoft reported 14 typosquatted npm packages published in a four-hour window. The packages targeted cloud and CI/CD secrets and executed through npm lifecycle hooks.

The victim does not need to import the package. Install is enough.

In plain English: the stolen material can include tokens used by build systems and cloud accounts. If those tokens work, an attacker may read private systems, push code, publish packages, or spend money before anyone notices.

how it got there

The attacker published packages with names close to legitimate ones. That is typosquatting: betting that someone mistypes, copies the wrong command, or trusts a familiar-looking name.

actual attack shape

npm install lookalike-package preinstall hook runs host and env get read payload steals cloud / CI / npm material
