Apr 2026 / fake package

Bitwarden CLI was fake. The package was not.

The name looked like something you might install in a hurry. That was the whole play. Borrow trust from a password-manager brand, get onto the machine, then look around.

Source: Palo Alto Networks

why it matters

Fake CLI packages are nasty because they meet developers where secrets already are. The command name sounds operational. The install path gets local access.

What gets stolen depends on what the machine exposes: environment variables, config files, session material, or credentials sitting in developer workflows. The consequence is simple: the attacker may get a working key without breaking into the provider directly.

how it got there

The attacker did not need to compromise the real Bitwarden CLI. They published a fake package that looked related enough to catch someone searching or copying an install command.
