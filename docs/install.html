Install

One command. Pick your setup below to see what comes next.

Piping to bash? Audit the script before running:

Step 1 — install

$ curl -fsSL worthless.sh | sh

Step 2 — what's your setup?

Run this once inside your project folder

$ cd your-project && worthless

No flags needed. worthless auto-detects your .env, asks which keys to protect, then starts the proxy daemon on port 8787. Takes about 90 seconds.

Your .env looks exactly the same after locking. The key format is preserved — an attacker who steals your file gets a dead shard that's indistinguishable from the real key.

Start the proxy, then register as an MCP server

$ cd your-project && worthless

Then add worthless as an MCP server in your project's .mcp.json: 

{
  "mcpServers": {
    "worthless": {
      "command": "worthless",
      "args": ["mcp"]
    }
  }
}

Claude Code picks up MCP servers automatically on next launch. How the split-key proxy works →

Start the proxy, then launch the MCP server

$ cd your-project && worthless

Worthless exposes an MCP server over stdio. Start it with:

$ worthless mcp

OpenClaw discovers it via ClawHub. How the split-key proxy works →

Open WSL 2 first — native Windows is not supported

The CLI actively refuses to start on native Windows (win32). Open a WSL 2 terminal, then run Step 1 above. Everything from there is identical to the solo dev flow.

Don't have WSL 2? Install it → — takes about 5 minutes.

Heads up: if you store your project under /mnt/c/ (the Windows drive), the proxy detects the WSL bridge and warns you about filesystem atomicity. Move the project to ~/ (the Linux side) for best results.

Verify it's working

$ worthless status

You should see PROTECTED next to each locked key, the proxy address, and live spend. If a key still shows ACTIVE, re-run worthless inside that project folder.
